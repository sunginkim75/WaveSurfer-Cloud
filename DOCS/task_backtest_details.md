# 백테스트 Task별 상세 조회 기능 설계 사양서

본 문서는 WS DASHBOARD에서 등록되어 운영 중인 Task별로 설정값을 불러와 서버사이드에서 백테스트를 실행하고, 그 상세 내역을 조회할 수 있도록 개발한 백테스트 연동 기능의 기술 명세를 설명합니다.

## 1. 배경 및 필요성
기존 시뮬레이터는 구글 스프레드시트에 값을 입력하고 수식이 계산될 때까지 대기한 후 데이터를 다시 불러오는 클라이언트-시트 간 강한 결합 구조를 가져, 응답이 느리고 불안정하며 구글 Oauth 로그인을 필요로 했습니다.
또한, 사용자가 운영 중인 여러 봇(Task)의 환경과 동일하게 시뮬레이션을 돌려보려면 매번 폼에 수동으로 파라미터를 입력해야 하는 번거로움이 있었습니다.

## 2. 주요 개선 사항
1. **서버 사이드 백테스트 엔진 구현 (`core/backtester.py`)**
   - 구글 시트 의존성 없이 `yfinance` API를 활용하여 역사적 주가 데이터와 QQQ 주간 RSI 데이터를 즉각 수집합니다.
   - 실제 매매 봇(`SurferBatchStrategy`)과 100% 동일한 로직의 모드 전환 및 복리 시스템을 파이썬으로 가속화하여 실행합니다.
2. **백테스트 REST API 노출 (`main.py` -> `/api/v1/backtest`)**
   - 백테스트 파라미터를 JSON으로 수신받아 시뮬레이션을 실행하고 요약 결과와 일자별 매매 상세 데이터를 프론트엔드로 반환합니다.
3. **대시보드 UI 개편 및 Task 바인딩 (`web/index.html`, `web/app.js`)**
   - 대시보드 하단에 **시뮬레이션** 탭을 신규 추가했습니다.
   - **🤖 운영 중인 Task 설정 불러오기** 드롭다운을 상단에 배치하여, 등록된 Task 선택 시 자동으로 입력 폼에 값이 채워지고 백테스트가 연동되어 실행되도록 구현했습니다.

## 3. 백테스트 시뮬레이션 매커니즘
- **모드 판정**: 주간 Wilder's RSI (14주봉) 기준, 전주 대비 상승/하락 트렌드와 RSI 값 구간에 따라 매 영업일의 모드(안전/공세)를 동적으로 결정합니다.
  - *하락 트렌드*: RSI > 65 or 40 < RSI < 50 or RSI 50 하향 돌파 시 -> `안전모드`
  - *상승 트렌드*: RSI 50 상향 돌파 or 50 < RSI < 60 or RSI < 35 시 -> `공세모드`
- **매도 판정**:
  - `LOC 매도`: 당일 종가가 매수 단가 기준 목표가(공세 `agg_sell_pct`, 안전 `safe_sell_pct`) 이상일 때 청산.
  - `MOC 청산`: 매수 이후 제한일수(공세 7일, 안전 30일)를 초과하여 경과했을 때 당일 종가로 즉시 청산.
- **매수 판정**:
  - `LOC 매수`: 전일 종가 기준 목표 매수가(공세 `agg_buy_pct`, 안전 `safe_buy_pct`) 이하로 당일 종가가 도달했을 때 1회 분할 금액 만큼 매수 진입.
- **복리 시스템**:
  - `update_period` (기본 10일) 거래일마다 최근 기간 동안의 실현손익(BFS)을 합산하여, 손익의 복리율(이익 `compounding_profit_rate`, 손실 `compounding_loss_rate`) 만큼 예수금 기준점(`last_compounding_cash`)을 가감 조절합니다.
