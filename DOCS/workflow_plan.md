# WaveSurfer 및 무한매수 워크플로우 및 라이프사이클 관리 계획서

본 문서는 WaveSurfer(WS) 및 무한매수(IB) 전략의 실제 한국 시간 기준 운영 워크플로우(Workflow), 작동 모드(실전/자동/수동/MOCK), 그리고 태스크 종료 및 사이클별 이력 관리 방안에 대한 설계 계획을 기술합니다.

---

## 1. 전략별 한국 시간 기준 데일리 워크플로우 (Daily Workflow)

미국 주식 시장의 스케줄에 맞춰 두 전략의 일일 운영 라이프사이클을 다음과 같이 정의합니다.

### A. WaveSurfer (WS) 워크플로우
WS는 장마감 기준 **보유 수량**의 정합성만 유지되면 로직에 무리가 없는 특징이 있습니다.
- **오전 06:10 (장 마감 직후 계좌 동기화)**:
  - 엔진이 실계좌 잔고를 조회하여 실제 보유 수량을 동기화합니다.
  - 전일 종가를 기반으로 당일 변동률 및 매매모드(안전/공세)를 판정합니다.
  - 로컬 DB에 당일자 데이터를 업데이트합니다.
- **오전 09:00 (당일 주문표 작성)**:
  - 당일 밤에 넣을 LOC 매수 목표가 및 목표 수량을 연산합니다.
  - 로컬 DB에 당일 '예정(Pending)' 주문 행을 생성합니다.
- **오후 18:00 (한국시간 저녁 프리장 오픈)**:
  - 예정된 LOC 매수 주문을 키움 OpenAPI 등으로 전송합니다 (실전 자동 모드인 경우).

### B. 무한매수 (IB) 워크플로우
무한매수는 누적 매입금과 회차(T)가 핵심이므로, **장마감 후 계좌의 예수금 및 평단가**를 기준으로 오늘 밤의 주문표를 매일 새로 작성해야 합니다.
- **오전 06:10 (장 마감 직후 계좌 동기화)**:
  - 엔진이 실계좌의 평단가, 보유수량, 예수금 잔고를 조회하여 DB에 동기화합니다.
  - 현재 누적 매입금과 진행 회차(T값)를 판정하여 로컬 DB를 업데이트합니다.
- **오전 09:00 (오늘 밤 주문표 작성)**:
  - 오늘 밤 실행할 주문을 연산합니다. (예: 평단 대비 +10% 지정가 매도, 평단 이하 LOC / 평단 이상 LOC 분할 매수 주문 등)
  - 로컬 DB에 당일 주문 예정 행을 저장합니다.
- **오후 18:00 (한국시간 저녁 프리장 오픈)**:
  - 연산된 지정가 매도 및 LOC 매수 주문을 시장으로 전송합니다.

## 2. 태스크 작동 모드 세분화 계획

태스크의 안정성과 테스트 유연성을 위해 **운영 모드**를 명확히 구분하여 데이터 모델 및 UI에 반영합니다.

| 구분 | 모드명 (Code) | 설명 |
| :--- | :--- | :--- |
| **실전 모드** | `REAL_AUTO` (실전 자동) | 키움 API를 연동하여 장마감 후 **자동 잔고 조회** 및 18:00 프리장에 **자동 주문 전송** 실행 |
| | `REAL_MANUAL` (실전 수동) | 실제 돈으로 투자하되, 주문은 사용자가 증권사 앱에서 **직접 실행**. 대시보드는 로컬 DB의 거래 내역을 기반으로 기록만 관리 |
| **모의 모드** | `MOCK_AUTO` (모의 자동) | 모의 투자 계좌 API를 연동하여 자동 거래 수행 (API 지원 시) |
| | `MOCK_VIRTUAL` (가상 테스트) | 실제 주문 없이 순수 알고리즘 테스트용 데이터만 로컬 DB에 누적 |

## 3. WaveSurfer (WS) 종료 및 이력 관리 방안

WS는 사이클 종료 없이 계속 복리로 자금을 불려 나가는 연속형 전략입니다. 따라서 유저가 자금을 일부 회수하거나 전략을 강제 종료하고 싶을 때의 관리법이 필요합니다.

### A. 태스크 상태값 정의
태스크에 `status` 필드를 도입합니다.
- `ACTIVE`: 현재 동작 중인 태스크
- `PAUSED`: 일시 중지 (주문 전송 중단, 상태 유지)
- `ARCHIVED` (보관): 전략 종료 및 자금 회수 완료 상태

### B. 보관(Archived) 프로세스 및 이력 저장
1. 유저가 대시보드에서 **[전략 종료 및 자금 회수]** 버튼을 누릅니다.
2. 엔진은 해당 태스크의 상태를 `ARCHIVED`로 즉시 변경합니다.
3. 종료 시점의 **최종 총자산, 예수금, 평가금액, 실현손익, 누적 수익률, 시작일, 종료일**을 고정(Freeze)하여 `task_history` 테이블에 기록합니다.
4. 대시보드상에서 해당 태스크에 대응하는 데이터 뷰를 보관(Archived) 상태로 전환하여 읽기 전용으로 잠금 처리합니다.
5. 유저는 언제든 보관된 과거 태스크들의 누적 수익 성과를 대시보드 '과거 이력 탭'에서 모아볼 수 있습니다.

## 4. 무한매수 (IB) 사이클별 수익률 관리 방안

무한매수는 평단 대비 목표가 전량 매도로 보유 수량이 `0주`가 되는 순간 **한 사이클이 종료**되는 특징이 있습니다. 사이클별 퍼포먼스 트래킹을 위한 데이터 모델을 제안합니다.

### A. 신규 데이터 모델: `infinite_buy_cycle`
사이클이 완료될 때마다 아래 정보를 담은 이력 테이블을 로컬 DB에 신설합니다.

```json
{
  "task_id": "task_ce7c4efe",
  "cycle_no": 1,
  "start_date": "2026-06-01",
  "end_date": "2026-07-10",
  "seed_amt": 10000,
  "max_invested_amt": 4500,
  "final_realized_profit": 450,
  "roi_on_seed": 4.5,
  "roi_on_max_invested": 10.0,
  "max_t_reached": 18
}
```
- **ROI on Seed (시드 대비 수익률)**: 전체 배정 시드(예: $10,000) 대비 수익률
- **ROI on Max Invested (실제 투자금 대비 수익률)**: 사이클 도중 실제 투입되었던 최대 누적 매입금(예: $4,500) 대비 수익률

### B. UI 및 대시보드 반영 방안
- **대시보드 모달**: 무한매수 태스크의 대조표 모달 상단에 **[사이클별 성과 요약]** 탭을 추가하여 역대 사이클들의 투입금 대비 수익률, 평균 진행 회차(T) 등을 차트 및 표로 시각화합니다.
- **대시보드 히스토리**: 태스크 상세 페이지 하단에 사이클 이력을 모아볼 수 있는 전용 그리드 뷰를 신설하여 이력을 축적합니다.

## 5. 실전 수동 모드 (REAL_MANUAL) 전용 제어 및 입력 사양

실전 수동 모드는 사용자가 직접 주문과 조회를 실행하므로, 시스템의 자동 스케줄러와 충돌하지 않으면서 개별적으로 액션을 취할 수 있는 **독립적인 수동 제어판(Control Panel)**이 필수적입니다.

### A. 대시보드 내 수동 제어 버튼 추가
수동 모드 태스크의 디테일 영역 혹은 대조표 모달에 아래의 독립 제어 버튼들을 제공합니다:
- **[수동 잔고 동기화]**: 키움 API 잔고 조회를 일회성으로 트리거하여 현재 보유량/평단가/예수금을 즉시 긁어와 로컬 DB를 강제 갱신합니다.
- **[주문표 재연산 및 갱신]**: 수동 기입되거나 동기화된 계좌 현황을 기준으로 오늘 밤 실행해야 할 주문 가이드(LOC 매수목표 등)를 즉각 재연산하여 표를 갱신합니다.

### B. 대조표 상세 거래 데이터 직접 기입 (UI Editor) 기능
사용자가 증권사 앱에서 매매를 처리한 후 대시보드상에서 직접 체결 값을 입력할 수 있도록 합니다.
- **거래 행 수정 모드**: 대조표 테이블의 각 행(특히 미체결 상태의 오늘/어제 거래 행)을 더블클릭하거나 수정 아이콘을 누르면 **[실제 체결 입력]** 팝업이 활성화됩니다.
- **기입 필드**: `실제 매수가 (buyPrice)`, `실제 매수량 (buyQty)`, `실제 매도가 (sellPrice)`, `실제 매도량 (sellQty)` 등.
- **연동 처리**: 사용자가 값을 입력하고 저장하면:
  1. 로컬 DB의 거래 기록에 값이 강제 주입(Force Override)됩니다.
  2. 완료된 체결 정보를 기준으로 자산 추이, 예수금, 평가금액, 실현손익이 실시간 재계산됩니다.
